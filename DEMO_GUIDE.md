# EasyMall 演示指南

## 1. 环境启动

### 启动 Redis
```bash
redis-server
```

### 启动 MySQL
确保 MySQL 服务已启动。

### 启动后端
```bash
cd easymall-java
# 启动用户端
cd easymall-web && mvn spring-boot:run
# 启动管理端（另开终端）
cd easymall-admin && mvn spring-boot:run
```

### 启动前端
```bash
# 用户端
cd easymall-web && npm run dev
# 管理端（另开终端）
cd easymall-admin-web && npm run dev
```

## 2. 默认数据准备

- 管理员账号：`admin` / `admin123`
- 已有测试商品：至少 2 件（请确保数据库中已有商品数据）
- 已有轮播图：至少 3 张（请确保数据库中已有轮播图）

## 3. 演示流程（完整闭环）

### 用户端演示（5 分钟）

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1. 注册用户 | 访问 `/register`，填写用户名、密码、手机号，点击注册 | 注册成功，跳转登录页 |
| 2. 用户登录 | 访问 `/login`，输入账号密码和验证码，点击登录 | 登录成功，跳转首页，右上角显示用户名 |
| 3. 浏览首页 | 查看轮播图、分类导航、热门商品 | 轮播图自动滚动，分类可点击展开 |
| 4. 商品详情 | 点击任意商品进入详情页，查看规格信息和 SKU 选项 | 商品信息正确展示，SKU 可点击切换 |
| 5. 加入购物车 | 选择规格和数量，点击「加入购物车」 | 提示成功，右上角购物车数量增加 |
| 6. 购物车管理 | 进入购物车，修改数量、选中/取消商品 | 金额实时更新，操作生效 |
| 7. 结算下单 | 点击「去结算」，填写收货信息（收货人、手机号、地址），提交订单 | 订单创建成功，跳转到订单列表 |
| 8. 查看订单 | 在订单列表中查看新订单，状态为「待付款」 | 订单信息完整正确 |
| 9. 模拟支付 | 在订单详情页点击「去支付」 | 订单状态变为「待发货」 |
| 10. 确认收货 | 点击「确认收货」 | 订单状态变为「已完成」 |

## 4. 常见问题

### 如果购物车加购失败
- 检查 Redis 是否启动（`redis-server`）
- 检查用户端后端日志（端口 6050）
- 检查商品是否存在且状态为启用

### 如果订单支付失败
- 确认订单状态为「待付款」（status=0）
- 确认订单归属为当前登录用户
- 检查后端日志是否有权限错误

### 如果管理端无法登录
- 确认账号密码为 `admin` / `admin123`
- 检查管理端后端日志（端口 6061）
- 检查 Redis 是否正确启动（验证码依赖 Redis）

### 如果页面无法访问
- 确认前端 dev server 已启动（端口 5173 / 5174）
- 确认后端服务已启动且端口正确
- 检查控制台是否有 CORS 或代理错误

## 5. 演示最佳实践

1. **提前准备**：在演示前先执行一次完整流程，确保所有环节正常
2. **清理数据**：提前清理旧的测试用户和订单数据，避免干扰
3. **商品准备**：确保数据库中有 3-5 件不同分类的商品，部分含 SKU
4. **多窗口展示**：使用两个浏览器窗口分别展示用户端和管理端，方便切换
5. **网络环境**：确保本地网络通畅，占位图片能正常加载（picsum.photos）
6. **数据回滚**：演示后可以恢复数据库到初始状态，便于重复演示
